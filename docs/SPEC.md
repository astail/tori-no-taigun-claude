# ゲーム仕様

リズム天国「鳥の大群」風リズムゲームの仕様書。

## 概要

- 群れの鳥（お手本）と同じタイミングで入力してステップを踏むリズムゲーム
- 曲は BPM 120・120拍 = **約60秒**
- ノーツ数 **50個**、満点 **100点**
- データは保存しない（リロードでリセット）

## 操作

| 環境 | 入力 |
| --- | --- |
| PC | ゲーム画面クリック / スペースキー |
| スマホ・タブレット | ゲーム画面タップ |

タイトル画面・リザルト画面のボタン以外の領域への入力はすべてゲーム入力として扱う。

## 判定と得点

入力時刻とノーツ時刻の差 Δt（ms）で判定する。

| 判定 | 条件 | 得点 | 演出 |
| --- | --- | --- | --- |
| ピタッ！（perfect） | \|Δt\| ≤ 60 | 2点 | 群れと同じステップ + キラン音 |
| まあまあ（good） | \|Δt\| ≤ 120 | 1点 | 群れと同じステップ |
| おっと…（bad / お手つき） | \|Δt\| ≤ 200 | 0点 | 失敗モーション（ノーツ消費） |
| おてつき！（stray / 空振り） | 上記の外 | 0点 | 失敗モーション（ノーツは残る） |
| ミス…（miss / 見逃し） | ノーツ後 200ms 経過 | 0点 | しょんぼりモーション |

- 入力は「最も近い未判定ノーツ」と照合する
- 50 ノーツ × 2点 = **100点満点**

## 譜面構成（BPM 120 / 1拍 = 0.5秒）

| 拍 | 内容 | ノーツ数 |
| --- | --- | --- |
| 0〜7 | イントロ（鳥が飛来・鳴き声の合図） | 0 |
| 8〜30 | 行進ステップ（2拍ごと） | 12 |
| 32〜37 | 掛け声 →「ビャー！ビャー！！」 | 2 |
| 40〜62 | 行進ステップ（2拍ごと） | 12 |
| 64〜69 | 掛け声 →「ビャー！ビャー！！」 | 2 |
| 72〜79 | 駆け足ステップ（毎拍） | 8 |
| 80〜85 | 掛け声 →「ビャー！ビャー！！」 | 2 |
| 88〜110 | 行進ステップ（2拍ごと） | 12 |
| 112〜 | 群れが飛び去る | 0 |
| 120 | 曲終了 → リザルト | — |

「掛け声」= ピー（1拍）ピャコ（1拍）ピャッコ（1拍）→ 1拍おいて ビャー！ビャー！！（各1拍・入力対象）。

## キャラクター（デフォルメ鳥）

Canvas 2D によるプログラム描画。アニメーション:

| モーション | 発生条件 |
| --- | --- |
| 飛来（フラップしながら降りてくる） | 曲開始〜約3.5秒 |
| 待機（上下ゆれ） | 通常時 |
| ステップ（ホップ + 左右リーン） | ノーツ成功 / 群れの自動動作 |
| ビャージャンプ（大ホップ + 羽ばたき + くちばし開き） | byaa ノーツ |
| 失敗（よろけ + 汗） | bad / stray |
| しょんぼり（縮む + 「…」） | miss |
| 鳴き声（くちばし開き） | 掛け声キュー |
| 飛び去り | 拍112以降 |

- 群れ: クリーム色 9羽（常にお手本どおり動く）
- プレイヤー: ピンク 1羽（前列中央・入力に反応して動く）

## サウンド（すべて Web Audio API で合成）

- **BGM**: 約60秒。C メジャーの行進曲風（キック・ベース・コード・ハット・メロディ + 終止和音）
- **鳴き声 SE**: 「ピー」「ピャコ」「ピャッコ」「ビャー！」を周波数スイープで合成
- **判定 SE**: perfect / good / bad / miss で別の音
- **ステップ音**: 群れのステップに合わせた「コッ」（リズムの目印）

## 画面構成

```
┌──────────────────────────┐
│  ゲーム画面（960×540）      │  空・雲・地面・鳥の群れ
│  （タイトル/リザルトはオーバーレイ） │
├──────────────────────────┤
│  タイミングチェック（960×150）│  DS下画面風パネル
└──────────────────────────┘
```

### タイミングチェックパネル（下画面）

- 中央線 = ピッタリ。ゾーン: 金（±60ms）/ 緑（±120ms）/ 灰（±200ms）
- 入力するたびに Δt の位置に刻み線が付き、約4秒かけてフェード
- 判定カウント（◎ピタッ！/ ○まあまあ / ✕ミス）とスコアを常時表示
- 直近の判定名をフラッシュ表示
